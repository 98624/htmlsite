Operators

  • i)Arithmetic operators -> eg +, -, *, /, %, ++, --, **

  • i)Arithmetic Operators

  • Consider A = 25, B = 7, Calculate C
    C = A + B it adds A & B, C = 32
    C = A - B it substracts B from A C = 18
    C = A * B it multiplies A and B C = 175
    C = A / B it devides A by B C = 3.571428571428571
    C = A % B it stores reminder of division of A by B, C = 4
    A++ it increments A by 1 A = 26
    B-- it decrements B by 1 B = 6
    3 ** 4 -> 81


    • ii)Relational operators -> eg ==, !=, >, <, >=, <=,

  • Consider A = 5, B = 10, C = 5
    d = '5'
    A == B False
    A == C True
    A == d True
    A === d False

    A >= C True
    C < B True
    B < C False

Functions

  • - A particular business logic is called as function.
    - Functions are used to reuse business logic.
    - We have following type of functions
    i) Named functions

  • i) Named functions

  • function with name is called as named functions
  • function function_name(parameters)
    {
    //business logic
    }
    //calling function
    function_name(arguments)

  • ii)Anonymous functions


  • Anonymous functions

  • - the function without name is called as anonymous function.
    - these are also called as arrow functions or callback functions.
    - we can represent arrow functions by '=>' arrow symbol.
    - Arrow functions introduced in ES6.
    - Arrow functions are more secured than named functions.
    - anonymous functions utilize heap memory efficiently.
    Syntax
    //function definition
    var / let variablename = (arguments) => {
    //business logic
    }
    //calling function
    variablename(parameters)

  • iii)Constructor functions
    - We can pass various types parameters to functions
    i) Rest Parameters
    ii)Optional Parameters
    iii)Default Parameters

Constructor Functions

  • - Constructor Function is used to create class like structure.
    - In constructor function everything (variables and functions) should start with 'this' keyword.
    - We can create object to constructor using 'new' keyword.
  • function class_one()
    {
    this.wish = "Welcome to contructor functions"
    }
    var obj = new class_one()
    console.log(obj.wish)

  • Prototype

  • Prototype is a property which adds members to constructor dynamically.
  • function class_one(){}
    class_one.prototype.wish = `Welcome to Prototype`
    console.log(new class_one().wish)

Passing parameters to functions

  • 1. Default parameters

  • - It allows named parameters to be intialysed with default values if no value or undefined is passed.
  • function fun_one(arg1 = 10, arg2 = "FullStack", arg3 = "Angular")
    {
    console.log(arg1, arg2, arg3)
    }

  • 2. Optional Parameters

  • - While calling a function no need to pass all arguments.
    - We can keep few arguments as optional.
    - Optional parameters instroduced in ES6.
  • function fun_one(arg1, arg2, arg3)
    {
    console.log(arg1, arg2, arg3)
    }

  • 3. Rest Parameters

  • - it is improved way to handle function parameters.
    - It allows us to represent infinite number of arguments as an array.
    - rest parameters represented by '...'
  • function fun_one(...arg)
    {
    console.log(arg)
    }

Promises

  • - Promises are used to handle asynchronous operations in Javascript.
    - Promises are special Javascript objects.
    - Promises have two states
    - Success (resolve)
    - Failure (reject)
    - Promises can be created using 'Promise' class.
    - 'Promise' is the predefined class in Javascript.
    - Promises can be consumed using 'then()'

  • //creating promise
    let my_promise = new Promise((resolve, reject)=>{
    resolve("Tomorrow i will be at home")
    })

  • //consume promise
    my_promise.then((posRes)=>{
    console.log(posRes)
    },(errRes)=>{
    console.log(errRes)
    })

  • async and await

  • - Netscape released above keywords in ES9.
    - Above keywords are used to increase code readability.
    - These keywords increase application performance.

  • async function my_fun()
    {
    let res = await my_promise
    console.log(res)
    }
